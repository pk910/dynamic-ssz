# Codegen Backward Compatibility Tests

This directory contains snapshots of generated SSZ code from each released version of the library that included the `codegen/tests` harness. These snapshots ensure that code generated by older versions of `dynssz-gen` continues to compile and produce correct results against the current library.

## How it works

Each `codegen_v*.tar.gz` archive contains the full test package (type definitions, generated code, and tests) as produced by that version's `dynssz-gen`. The generated code is **not** regenerated -- the point is to verify the library remains backward compatible with previously generated code.

During CI, the archives are unpacked and the tests are run alongside the regular test suite via `go test ./...`.

## Local development

To unpack the archives for local testing:

```sh
make unpack-compat-tests
```

The unpacked `v*/` directories are gitignored. Only the `.tar.gz` archives are tracked.

## Adding a new version

New archives are created automatically by the release workflow (`prepare-release.sh`). It builds `dynssz-gen` from the release source, generates the test code, and packages everything into a new `codegen_v*.tar.gz`.
